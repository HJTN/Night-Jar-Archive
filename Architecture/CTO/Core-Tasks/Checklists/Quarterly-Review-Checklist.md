# Quarterly Review Checklist

> 분기별 엔지니어링 목표 달성 여부와 다음 분기 방향을 점검하는 항목

## 1. OKR 점검

- [ ] 각 팀의 [[OKR]] Key Result 달성률을 수집했는가?
- [ ] 달성률 70% 미만인 KR의 원인을 분석했는가?
- [ ] 달성률 100%인 KR이 너무 많다면 목표가 너무 쉬웠던 것은 아닌가?
- [ ] 다음 분기 OKR 초안을 팀 리더와 함께 작성했는가?

## 2. 기술 로드맵 점검

- [ ] 이번 분기 로드맵 이니셔티브 진행 상황을 확인했는가?
- [ ] 지연된 항목의 원인(예: 요구사항 변경, 기술 부채, 리소스 부족)을 파악했는가?
- [ ] [[Technical-Debt]] 해소 목표 대비 실제 진행률을 확인했는가?
- [ ] 다음 분기 우선순위를 비즈니스 전략과 재정렬했는가?

## 3. 시스템 및 인프라 건강도

- [ ] SLO/SLA 달성률을 검토했는가?
- [ ] 이번 분기 주요 장애 및 포스트모템 내용을 리뷰했는가?
- [ ] 보안 취약점 패치 현황을 확인했는가?
- [ ] 클라우드 비용 트렌드를 검토하고 이상 항목을 확인했는가?

## 4. 팀 및 채용

- [ ] 팀 eNPS(만족도)를 측정했는가?
- [ ] 이직자 Exit Interview 인사이트를 검토했는가?
- [ ] [[Engineering-Hiring-Process]] 파이프라인 현황을 확인했는가?
- [ ] 다음 분기 채용 목표를 [[OKR]]에 반영했는가?

## 5. 기술 역량 및 학습

- [ ] 팀이 [[Tech-Radar]]의 Trial·Assess 항목을 실제로 실험했는가?
- [ ] 기술 교육·컨퍼런스 참가·사내 스터디 현황을 확인했는가?
- [ ] 새로운 기술 후보를 수집해 다음 [[Tech-Radar-Review-Process]]에 준비했는가?

## 6. 경영진 보고 준비

- [ ] 이번 분기 엔지니어링 주요 성과를 비즈니스 임팩트 언어로 정리했는가?
- [ ] 다음 분기 투자 요청 사항(채용, 인프라, 도구)을 근거와 함께 준비했는가?
- [ ] [[Roadmunk]] 또는 로드맵 도구로 시각화된 보고 자료를 준비했는가?

## Notes

- 분기 리뷰는 비판이 아닌 학습의 장. 실패도 솔직하게 공유
- [[Notion]] 또는 [[Confluence]]에 분기별 리뷰 문서를 축적해 연간 흐름 파악
- 리뷰 후 액션 아이템은 [[Jira]]에 등록해 담당자·기한 지정
