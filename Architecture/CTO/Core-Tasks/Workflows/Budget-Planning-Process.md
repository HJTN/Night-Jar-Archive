# Budget Planning Process

> 연간 엔지니어링 기술 예산을 기획하고 경영진 승인을 받는 절차

## Overview

CTO가 연간 기술 투자 계획(채용·인프라·도구·교육)을 수립하고 CFO·CEO와 합의해 예산을 확정하는 프로세스. 보통 Q3~Q4에 진행하며, 다음 연도 [[Technology-Strategy-Planning|기술 전략]]과 연동된다.

## Steps

1. **현황 분석 (8~10주 전)** — 올해 실제 집행 예산 대비 계획 검토. 초과·절감 항목 원인 분석
2. **기술 전략 인풋 반영** — [[Technology-Strategy-Planning]] 결과물 기반으로 필요 투자 항목 도출
3. **항목별 예산 산정**
   - 채용: 목표 헤드카운트 × 평균 TC + 채용 대행 비용
   - 인프라·클라우드: 올해 비용 × 성장률 ± 최적화 절감분 → [[FinOps]] 데이터 활용
   - 도구·라이선스: 갱신 항목 + 신규 도입 예정 도구
   - 교육·컨퍼런스: 1인당 예산 × 팀 인원
4. **CapEx vs OpEx 분류** — 재무팀과 협의해 각 항목을 자본 지출 / 운영 비용으로 분류
5. **ROI 근거 작성** — 주요 투자 항목별 기대 효과를 비즈니스 임팩트로 표현

   | 투자 항목 | ROI 표현 예시 |
   |---|---|
   | DevOps 도구 $X | 배포 빈도 2배, 엔지니어 연간 X시간 절약 |
   | 보안 도구 $X | SOC 2 인증 취득 → 엔터프라이즈 영업 가능 |
   | 채용 N명 | 기술 로드맵 Y분기 단축 가능 |

6. **CFO 사전 협의** — 예산 초안을 CFO와 공유. 재무 제약·우선순위 파악
7. **트레이드오프 준비** — 예산 삭감 시나리오별 영향도 분석 (Must Have / Nice to Have 분류)
8. **경영진 승인** — 최종 예산안 발표. 투자 우선순위 합의
9. **분기별 추적** — [[Quarterly-Review-Checklist]] 기반으로 예산 집행 현황 점검

## 예산 구성 항목

| 카테고리 | 세부 항목 | 유형 |
|---|---|:---:|
| **인력** | 신규 채용 TC, 채용 대행 수수료 | OpEx |
| **클라우드 인프라** | AWS/GCP/Azure 비용, 예약 인스턴스 | OpEx |
| **SaaS 도구** | Datadog, GitHub, Jira, Notion 등 | OpEx |
| **보안 & 컴플라이언스** | 보안 감사, 침투 테스트, 인증 비용 | OpEx |
| **교육 & 성장** | 컨퍼런스, 온라인 강의, 도서 지원 | OpEx |
| **하드웨어 & 장비** | 개발 장비, 사무 인프라 | CapEx |
| **R&D / 혁신** | PoC 프로젝트, AI 도구 실험 | OpEx/CapEx |

## FinOps 연계

- 클라우드 비용은 [[FinOps]] 원칙에 따라 팀·서비스별 비용 배분(Chargeback) 체계 적용
- Reserved Instance / Savings Plan으로 예약 구매해 20~40% 절감 가능
- [[Datadog]] 또는 [[Grafana]]로 실시간 비용 트렌드 모니터링

## Inputs

- 올해 실집행 예산 데이터
- 기술 로드맵 및 채용 계획
- 클라우드 비용 리포트 ([[FinOps]])
- 시장 보상 데이터 (채용 TC 산정용)

## Outputs

- 연간 기술 예산 계획서 (항목별 근거 포함)
- CapEx/OpEx 분류표
- 예산 삭감 시나리오별 트레이드오프 문서

## Notes

- 예산은 전략의 언어. **무엇에 돈을 쓰는가**가 우선순위를 보여준다
- "기술 부채 상환"을 예산 항목으로 명시하면 비즈니스와 협상하기 쉬워진다 → [[Technical-Debt]]
- [[Confluence]]에 연도별 예산 계획서를 보관해 히스토리 유지
