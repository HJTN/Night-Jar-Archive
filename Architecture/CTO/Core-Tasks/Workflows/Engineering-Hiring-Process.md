# Engineering Hiring Process

> 포지션 정의부터 온보딩까지 엔지니어 채용 전체 흐름

## Overview

CTO가 주도하는 엔지니어링 채용 프로세스. 단순한 기술 검증을 넘어 [[Engineering-Culture]] 적합성, 성장 가능성, 팀 다이나믹스를 종합 평가한다. 채용은 CTO가 할 수 있는 가장 큰 레버리지 투자다.

## Steps

1. **포지션 정의** — 비즈니스 목표·기술 로드맵 기반으로 필요한 역할·스킬셋·시니어리티 결정. JD(Job Description) 작성
2. **채용 기준 정렬** — 면접관 팀과 평가 기준 합의. 기술 역량·문제해결·커뮤니케이션·[[Engineering-Culture]] 적합성 항목 정의
3. **소싱** — LinkedIn, 컨퍼런스 네트워킹, 사내 추천, 오픈소스 기여자 탐색. Passive 후보자에게 직접 아웃리치
4. **이력서 스크리닝** — CTO 또는 시니어가 1차 필터링. 기술 키워드보다 프로젝트 임팩트·성장 궤적에 집중
5. **1차 인터뷰 (30~45분)** — 채용 담당자 또는 엔지니어링 매니저. 동기·경험·커뮤니케이션 평가
6. **기술 과제 / 코딩 인터뷰** — 실무 중심의 과제. 알고리즘 퍼즐보다 실제 문제 해결 방식 평가
7. **시스템 설계 인터뷰 (시니어 이상)** — 확장성·트레이드오프·[[Technical-Debt]] 인식 수준 평가
8. **문화 인터뷰** — CTO 또는 팀 리더 진행. 실패 경험·협업 방식·학습 태도 심층 탐색
9. **레퍼런스 체크** — 이전 매니저·동료 2~3명. 강점·개선 영역·팀 기여도 확인
10. **오퍼 및 협상** — 시장 데이터 기반 보상 설계. 주식·성장 기회·문화 강점 설명
11. **온보딩 설계** — 30·60·90일 플랜 준비. 버디 프로그램, 첫 기여 이슈(Good First Issue) 지정

## Inputs

- 기술 로드맵 및 팀 갭 분석
- 채용 예산
- 역할별 평가 루브릭

## Outputs

- 채용 완료 엔지니어 (온보딩 중)
- 인터뷰 피드백 문서 (채용 여부와 무관하게 보관)
- 채용 프로세스 개선 인사이트

## Notes

- **다양성**: 동질적인 팀은 혁신이 느리다. 배경·경험·사고방식의 다양성을 의도적으로 추구
- **속도 vs 품질**: 채용을 서두르면 문화 문제가 생긴다. 포지션 공석이 길어도 기준을 낮추지 않음
- [[OKR]]로 채용 목표 (예: Q2까지 백엔드 3명 합류) 추적
- [[Jira]] 또는 ATS(Applicant Tracking System)로 파이프라인 관리
