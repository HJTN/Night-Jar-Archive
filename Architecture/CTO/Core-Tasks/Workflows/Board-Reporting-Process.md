# Board Reporting Process

> 이사회·경영진에게 기술 전략과 성과를 보고하는 절차

## Overview

CTO가 이사회(Board) 또는 경영진(C-suite)에게 기술 현황·전략·투자 요청을 보고하는 프로세스. 기술 용어 대신 비즈니스 임팩트 언어로 소통하는 것이 핵심이다. 보통 분기별 1회 진행하며, 연간 기술 전략 발표는 별도 세션으로 진행한다.

## Steps

1. **보고 목적 정의 (1주 전)** — 이번 보고의 핵심 메시지 3가지 결정. 정보 공유 / 의사결정 요청 / 투자 승인 중 무엇인지 명확히
2. **데이터 수집** — [[OKR]] 달성률, SLO 현황, 인프라 비용, 채용 현황, 주요 이니셔티브 진행률 취합 → [[Quarterly-Review-Checklist]] 활용
3. **번역 작업** — 기술 지표 → 비즈니스 임팩트 언어로 변환

   | 기술 언어 | 비즈니스 언어 |
   |---|---|
   | 배포 빈도 2배 증가 | 신기능 출시 주기 절반으로 단축 |
   | 인프라 비용 20% 절감 | 연간 $X 절약, 마진율 개선 |
   | SLO 99.9% 달성 | 연간 다운타임 8시간 이내 보장 |

4. **스토리라인 구성** — 과거(성과) → 현재(현황·리스크) → 미래(전략·투자 요청) 흐름
5. **슬라이드 작성** — [[Roadmunk]] 또는 PowerPoint. 슬라이드당 1개 메시지 원칙
6. **사전 리뷰** — CFO·CEO와 내용 공유. 예상 질문 준비 (재무 임팩트·리스크 위주)
7. **보고 진행** — 10~15분 발표 + Q&A. 기술 세부사항은 Appendix로 분리
8. **후속 조치** — 결정 사항·액션 아이템을 [[Confluence]]에 문서화. 담당자·기한 지정

## 보고 슬라이드 구조

| 슬라이드 | 내용 |
|:---:|---|
| 1 | Executive Summary (핵심 3줄 요약) |
| 2 | 이전 분기 주요 성과 (비즈니스 임팩트 중심) |
| 3 | 현재 리스크 및 현황 |
| 4 | 다음 분기 주요 이니셔티브 |
| 5 | 투자 요청 및 근거 |
| 6 | 기술 로드맵 타임라인 (고수준) |
| Appendix | 상세 지표, 기술 세부 내용 |

## 핵심 원칙

- **기술 용어 배제** — 이사회에 "쿠버네티스", "마이크로서비스"는 쓰지 않는다
- **나쁜 소식도 먼저** — 리스크를 숨기면 신뢰를 잃는다. 문제와 대응책을 함께 제시
- **숫자로 말하기** — 모든 주장에 데이터를 붙인다
- **의사결정 요청을 명확히** — "승인 요청", "정보 공유", "의견 요청" 중 무엇인지 명시

## Inputs

- 분기 [[OKR]] 달성 데이터
- 인프라 비용 리포트 ([[FinOps]])
- 채용 현황 파이프라인
- 주요 리스크·이슈 목록

## Outputs

- 이사회 보고 슬라이드 ([[Confluence]] 버전 보관)
- 결정 사항 기록
- 후속 액션 아이템 (담당자·기한 포함)

## Notes

- [[Quarterly-Review-Checklist]]로 데이터를 미리 준비해두면 보고 작성 시간을 단축할 수 있음
- 이사회 구성원별 관심사가 다를 수 있음 (투자자 → ROI, 사외이사 → 리스크 관리, CEO → 실행 속도)
- 보고 자료는 [[Confluence]]에 연도·분기별 버전 보관해 감사 이력 유지
