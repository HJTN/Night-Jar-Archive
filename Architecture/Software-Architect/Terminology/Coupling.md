# 결합도 (Coupling)

## Definition

모듈(컴포넌트, 서비스) 간의 의존 관계 강도. 결합도는 **낮을수록 좋다**.

높은 결합도 = 변경 시 연쇄 영향 → 리팩터링·테스트·배포 어려움.

## Context / When to Use

모듈 간 인터페이스를 설계하거나 의존성 방향을 결정할 때 판단 기준이 되는 개념.

## 결합도 유형 (높은 순 → 낮은 순)

| 유형 | 설명 | 예시 |
|---|---|---|
| **콘텐트 (Content)** | 다른 모듈의 내부 데이터 직접 수정 | 전역 변수 직접 수정 |
| **공통 (Common)** | 전역 공유 데이터에 의존 | 전역 Config 객체 |
| **외부 (External)** | 외부 포맷·프로토콜에 하드코딩 의존 | CSV 파싱 하드코딩 |
| **제어 (Control)** | 제어 플래그를 인자로 전달 | `process(flag: bool)` |
| **스탬프 (Stamp)** | 필요 이상의 데이터 구조 전달 | 전체 User 객체 전달 |
| **데이터 (Data)** | 필요한 데이터만 파라미터로 전달 | `process(id, name)` |
| **메시지 (Message)** | 인터페이스·이벤트로만 소통 | 이벤트 버스, 인터페이스 |

## 낮은 결합도 달성 방법

| 방법 | 설명 |
|---|---|
| **인터페이스 추상화** | 구현체가 아닌 인터페이스에 의존 (DIP) |
| **이벤트 기반 통신** | 직접 호출 대신 이벤트/메시지로 소통 |
| **의존성 주입 (DI)** | 의존성을 외부에서 주입 (IoC 컨테이너) |
| **API Gateway** | 서비스 간 직접 의존 대신 API Gateway 경유 |
| **데이터 파라미터화** | 필요한 데이터만 인자로 전달 |

## 결합도 vs 응집도

"높은 응집도 + 낮은 결합도"는 좋은 설계의 핵심 원칙이다.

- **[[Cohesion|응집도]]**: 모듈 내부 관련도 — 높을수록 좋음
- **결합도**: 모듈 간 의존도 — 낮을수록 좋음
