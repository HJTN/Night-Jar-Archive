# RFP (Request for Proposal, 제안 요청서)

## Definition

고객(발주기관)이 특정 시스템·서비스 구축을 위해 공급자(벤더·컨설팅사)에게 **솔루션 제안을 공식 요청하는 문서**.

RFP에 응답하는 문서를 제안서(Proposal)라 하며, 솔루션 아키텍트는 기술 제안 파트를 주도적으로 작성한다.

## Context / When to Use

- 기업이 외부 벤더에게 IT 시스템 구축을 의뢰할 때
- 정부·공공기관이 IT 사업을 발주할 때
- 클라우드 마이그레이션, 플랫폼 구축 등 대규모 프로젝트

## RFP 구성 요소

| 섹션 | 내용 |
|---|---|
| **개요** | 사업 목적, 배경, 범위 |
| **요구사항** | 기능 요구사항 (FR), [[NFR|비기능 요구사항(NFR)]] |
| **일정** | 제안서 제출 기한, 사업 일정 |
| **평가 기준** | 기술점수·가격점수 비율, 평가 항목 |
| **제출 형식** | 페이지 수 제한, 파일 형식, 제출 방법 |
| **계약 조건** | SLA, 보증 기간, 페널티 조항 |

## RFP → 제안서 흐름

```
RFP 수신
→ RFP 분석 (요구사항 파악, 평가 기준 확인)
→ 참여 여부 결정 (Bid/No-Bid)
→ 솔루션 설계 ([[Solution-Design-Process]])
→ 제안서 작성
→ [[RFP-Response-Checklist]] 검토
→ 제출
→ 발표 (Q&A)
→ 수주 or 탈락
```

## Bid/No-Bid 판단 기준

| 판단 요소 | 고려사항 |
|---|---|
| **기술 역량** | 요구 기술을 보유하고 있는가? |
| **수익성** | 예상 수주 금액이 투입 비용 대비 수익적인가? |
| **리스크** | 요구사항이 불명확하거나 일정이 비현실적인가? |
| **레퍼런스** | 유사 사례가 있는가? |
| **경쟁 상황** | 경쟁사 대비 차별점이 있는가? |
