# Solution Architect

> 고객 요구사항 기반 기술 솔루션 설계 및 제안

## Overview

Solution Architect는 고객(내부 또는 외부)의 비즈니스 문제를 기술 솔루션으로 해결하는 설계 전문가다. 요구사항 분석부터 솔루션 제안·설계·PoC 검증까지를 담당하며, 기술과 비즈니스를 잇는 브리지 역할을 한다. 벤더 독립적인 관점에서 다양한 기술 옵션을 비교해 최적 솔루션을 제안한다.

**Solution Architect vs Software Architect:**
- **Solution Architect** — 특정 고객·프로젝트의 요구사항에 맞는 솔루션 탐색·제안. 엔터프라이즈·클라우드 아키텍처와 통합·마이그레이션에 강점
- **Software Architect** — 개발 조직 내에서 시스템의 기술 구조를 코드 레벨까지 설계

**주요 활동 맥락:**
- SI/컨설팅 회사 → 고객 프로젝트 솔루션 설계 및 RFP 대응
- 클라우드 벤더(AWS, GCP, Azure) → 고객의 클라우드 전환 아키텍처 설계
- 대기업 내부 → 엔터프라이즈 시스템 통합·현대화 아키텍처

## Key Responsibilities

- **솔루션 설계** — 요구사항 분석 → 기술 솔루션 아키텍처 설계 → 제안서 작성
- **[[RFP]] 대응** — 고객 RFP 분석, 기술 제안서(Technical Proposal) 작성
- **[[PoC]] 실행** — 핵심 기술 가설을 검증하는 개념 증명 실험
- **[[Enterprise-Architecture|EA]] 기반 설계** — TOGAF 프레임워크 등 EA 맥락에서 솔루션 정의
- **이해관계자 커뮤니케이션** — 비기술 고객에게 솔루션 설명·설득
- **벤더·기술 평가** — 다양한 솔루션 옵션의 TCO·장단점 비교
- **클라우드 마이그레이션 설계** — 6Rs 전략 적용, 마이그레이션 로드맵
