# Confluence (Solution Architect 관점)

> Atlassian의 팀 위키·문서 협업 플랫폼

## Overview

솔루션 아키텍트에게 Confluence는 솔루션 문서·제안서·리뷰 결과를 이해관계자와 공유하고 관리하는 **공식 지식 저장소** 역할을 한다. 특히 내부 고객(사업부)과의 협업에서 설계 산출물을 투명하게 공유하는 데 사용한다.

## Solution Architect 주요 활용

| 활용 | 내용 |
|---|---|
| **SAD 게시** | Solution Architecture Document 공식 게시 |
| **[[RFP]] 분석 노트** | RFP 요구사항 분석 및 팀 논의 기록 |
| **[[PoC]] 결과 보고** | PoC 실험 결과 및 권고안 공유 |
| **리뷰 결과** | [[Solution-Review-Checklist]] 결과 및 액션 아이템 기록 |
| **레퍼런스 아키텍처** | 조직 내 검증된 아키텍처 패턴 라이브러리 |

## Key Features

| 기능 | 설명 |
|---|---|
| **Space** | 팀·프로젝트별 독립된 위키 공간 |
| **Page Templates** | SAD 템플릿, PoC 보고서 템플릿 적용 |
| **다이어그램 임베드** | [[Draw-io]] / [[Lucidchart]] 다이어그램 직접 삽입 |
| **Compare** | 페이지 버전 간 변경 내용 비교 |
| **권한 관리** | 페이지별 閲覧·편집 권한 설정 |

## Jira 연동

- 아키텍처 리뷰 액션 아이템을 Jira 이슈로 등록
- 프로젝트 진행 상황을 Confluence 대시보드로 요약

## Tips & Shortcuts

- `Ctrl + K` — 페이지 링크 삽입
- `/draw.io` — 다이어그램 블록 삽입
- `/table` — 표 삽입
- 페이지 레이블(Label)로 SAD / PoC / Review 분류
- 페이지 감시(Watch) → 변경 시 이메일 알림
