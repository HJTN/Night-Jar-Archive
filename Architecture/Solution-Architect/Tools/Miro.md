# Miro (Solution Architect 관점)

> 온라인 화이트보드·워크숍 협업 도구

## Overview

솔루션 아키텍트에게 Miro는 **고객 Discovery 워크숍**과 **아키텍처 브레인스토밍**에 특히 유용하다. 정식 다이어그램 도구([[Draw-io]], [[Lucidchart]]) 이전 단계에서 아이디어를 빠르게 시각화하고 고객과 실시간으로 공유할 때 사용한다.

## Solution Architect 주요 활용

| 활용 | 설명 |
|---|---|
| **Discovery 워크숍** | 고객과 현황·문제·목표를 함께 시각화 |
| **아키텍처 브레인스토밍** | 초기 솔루션 아이디어 스케치 |
| **[[Trade-off]] 분석 보드** | 대안 비교 및 트레이드오프 시각화 |
| **[[RFP]] 분석** | RFP 요구사항을 팀이 함께 분해·분류 |
| **이해관계자 맵** | 고객 조직 내 이해관계자 구조 시각화 |

## Key Features

| 기능 | 설명 |
|---|---|
| **실시간 협업** | 원격 참가자와 동시 작업 가능 |
| **다양한 템플릿** | 워크숍, SWOT, 플로우차트, 마인드맵 등 |
| **스티커 노트** | 브레인스토밍 및 포스트잇 스타일 협업 |
| **타이머** | 워크숍 시간 관리 |
| **투표** | 팀 의견 수렴 (우선순위 결정) |

## vs 다른 도구 비교

| 상황 | 추천 도구 |
|---|---|
| 고객 워크숍, 초기 스케치 | **Miro** |
| 정식 아키텍처 다이어그램 | [[Draw-io]] 또는 [[Lucidchart]] |
| 아키텍처 문서 게시 | [[Confluence]] |

## Tips & Shortcuts

- `Ctrl + /` — 단축키 목록 보기
- `S` — 스티커 노트 추가
- `T` — 텍스트 추가
- `F` — 프레임 추가 (영역 구분, 각 토픽을 프레임으로 나누기)
- `Ctrl + Shift + K` — 선택 요소 잠금 (배경 고정)
