# AWS Architecture Icons & Tools

> AWS 공식 아키텍처 다이어그램 아이콘 세트

## Overview

AWS가 제공하는 공식 아키텍처 다이어그램 아이콘 세트와 관련 도구들. 솔루션 아키텍트가 AWS 기반 아키텍처를 시각화할 때 표준 아이콘을 사용해 전문성과 일관성을 높인다.

[[Draw-io]], [[Lucidchart]], PowerPoint에 AWS 아이콘을 임포트해 사용하거나, AWS 전용 도구(CloudFormation Designer, Architecture Center)를 활용한다.

## Key Features

| 도구/리소스 | 설명 |
|---|---|
| **AWS Icon Set** | EC2, S3, RDS, Lambda 등 모든 AWS 서비스 공식 아이콘 |
| **AWS Architecture Center** | 검증된 솔루션별 레퍼런스 아키텍처 예시 모음 |
| **AWS Well-Architected Tool** | 6대 기둥 기반 아키텍처 검토 도구 (콘솔 내) |
| **CloudFormation Designer** | IaC 코드를 다이어그램으로 시각화 |
| **AWS Application Composer** | 서버리스 아키텍처 시각적 설계 도구 |

## AWS 6대 Well-Architected 기둥

솔루션 설계 시 아키텍처를 평가하는 기준:

| 기둥 | 핵심 질문 |
|---|---|
| **안정성 (Reliability)** | 장애를 자동 복구할 수 있는가? |
| **성능 효율성 (Performance Efficiency)** | 리소스를 효율적으로 사용하는가? |
| **보안 (Security)** | 데이터와 시스템을 보호하는가? |
| **비용 최적화 (Cost Optimization)** | 불필요한 비용을 제거했는가? |
| **운영 우수성 (Operational Excellence)** | 운영과 모니터링이 자동화되어 있는가? |
| **지속가능성 (Sustainability)** | 환경 영향을 최소화하는가? |

## Common Usage (Solution Architect)

- **RFP 응답 다이어그램**: AWS 공식 아이콘으로 신뢰성 있는 다이어그램 작성
- **아키텍처 리뷰**: Well-Architected Framework 기반 설계 검토
- **레퍼런스 아키텍처 참조**: Architecture Center에서 유사 사례 탐색
- **비용 추산**: AWS Pricing Calculator와 연계

## Installation & Setup

1. [AWS Architecture Icons](https://aws.amazon.com/architecture/icons/) 다운로드
2. Draw.io: `Edit > XML` 또는 Shape Library Import
3. PowerPoint: 아이콘 파일(.pptx) 다운로드 후 슬라이드에 삽입

## Tips & Shortcuts

- 서비스 아이콘(Service Icon) vs 리소스 아이콘(Resource Icon) 구분 — 다이어그램 레벨에 따라 선택
- AWS 색상 코드 준수: 컴퓨팅(주황), 스토리지(초록), 데이터베이스(파랑), 네트워킹(보라)
- Architecture Center에서 "자신의 상황과 가장 유사한" 레퍼런스 아키텍처 찾기
