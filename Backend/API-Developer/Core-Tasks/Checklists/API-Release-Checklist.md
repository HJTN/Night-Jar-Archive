# API Release Checklist

> API 배포 전 확인해야 할 항목

## Checklist

### 버전 및 스펙
- [ ] API 버전이 URI에 명시됐는가 (`/v1/`, `/v2/`)
- [ ] [[Versioning|Breaking Change]]가 있다면 새 버전으로 분리됐는가
- [ ] OpenAPI 스펙 문서가 최신 구현과 일치하는가
- [ ] [[Swagger]] UI에서 모든 엔드포인트가 정상 표시되는가
- [ ] Changelog에 변경 사항이 기록됐는가

### 인증 / 보안
- [ ] 모든 보호 엔드포인트에 인증 미들웨어가 적용됐는가
- [ ] HTTPS 강제 설정이 됐는가 (HTTP → HTTPS 리다이렉트)
- [ ] CORS 정책이 허용된 Origin만 명시됐는가 (`*` 금지)
- [ ] [[Rate-Limiting]] 설정이 적용됐는가
- [ ] 민감 정보가 로그에 기록되지 않는가 (토큰, 카드번호)

### 입력 검증
- [ ] 모든 입력값에 서버 측 유효성 검증이 있는가
- [ ] SQL Injection, XSS 방어 처리가 됐는가
- [ ] 파일 업로드 API의 크기·타입 제한이 있는가

### 에러 응답
- [ ] 에러 응답 형식이 전체 API에서 통일됐는가
- [ ] 서버 오류(500) 응답에 내부 스택 트레이스가 노출되지 않는가
- [ ] 알려진 예외 케이스가 적절한 상태 코드로 처리됐는가

### 성능
- [ ] 목록 조회 API에 [[Pagination]]이 적용됐는가
- [ ] 페이지 크기 상한값이 설정됐는가 (예: 최대 100건)
- [ ] 자주 조회되는 엔드포인트에 캐시 전략이 있는가
- [ ] DB 쿼리에 필요한 인덱스가 생성됐는가
- [ ] 부하 테스트(k6, nGrinder)를 통해 목표 TPS를 검증했는가

### 하위 호환성
- [ ] 기존 클라이언트가 사용 중인 필드를 제거하지 않았는가
- [ ] 기존 응답 필드의 타입을 변경하지 않았는가
- [ ] Deprecated 표시가 필요한 필드에 안내가 추가됐는가

### 테스트
- [ ] 단위 테스트가 통과하는가 (`./gradlew test`)
- [ ] 통합 테스트가 통과하는가
- [ ] CI/CD 파이프라인 빌드·테스트가 녹색인가
- [ ] [[Postman]] Collection으로 주요 시나리오 테스트를 완료했는가

### 모니터링 및 운영
- [ ] 새 엔드포인트에 대한 응답 시간·에러율 모니터링이 설정됐는가
- [ ] 이상 트래픽 감지 알림이 설정됐는가
- [ ] 배포 후 롤백 계획이 준비됐는가

## Notes

- 이 체크리스트는 배포 전날 또는 배포 직전에 확인
- 팀원과 함께 페어 체크를 하면 누락 항목을 줄일 수 있음
- 설계 단계 점검은 [[API-Review-Checklist]] 사용
