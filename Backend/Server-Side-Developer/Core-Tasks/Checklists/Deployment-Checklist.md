# Deployment Checklist

> 서버 애플리케이션 배포 전 확인해야 할 항목

## 배포 전 준비

### 코드 및 테스트
- [ ] 모든 테스트(단위/통합)가 CI에서 통과했는가?
- [ ] 코드 리뷰가 완료되고 승인되었는가?
- [ ] 배포할 버전이 스테이징 환경에서 검증되었는가?

### DB 마이그레이션
- [ ] 마이그레이션 스크립트가 스테이징 환경에서 먼저 실행되었는가?
- [ ] 마이그레이션 실패 시 롤백 스크립트가 준비되어 있는가?
- [ ] 컬럼 삭제 등 비가역적 변경인 경우 충분한 검토가 이루어졌는가?
- [ ] 대규모 마이그레이션(수백만 행 변경)의 경우 서비스 영향도를 파악했는가?
- [ ] 배포 전 마이그레이션을 적용할지 (migrate-before-deploy) 또는 하위 호환 유지하며 배포 후 정리할지 전략이 결정되어 있는가?

### 환경변수 및 시크릿
- [ ] 운영 환경에 필요한 환경변수가 모두 설정되어 있는가?
- [ ] 신규 추가된 환경변수/시크릿이 배포 환경에 반영되었는가?
- [ ] 시크릿(DB 비밀번호, API 키 등)이 코드에 하드코딩되지 않았는가?

### 기능 플래그 (Feature Flag)
- [ ] 미완성이거나 점진적 롤아웃이 필요한 기능에 기능 플래그가 설정되어 있는가?
- [ ] 기능 플래그 초기 상태(ON/OFF)가 의도한 대로 설정되어 있는가?

## 배포 중

### 헬스체크 확인
- [ ] `/health` 또는 `/actuator/health` 엔드포인트가 정상 응답하는가?
- [ ] 로드밸런서의 헬스체크가 새 인스턴스를 정상으로 인식하는가?

### 순차 배포 (Rolling/Blue-Green)
- [ ] 한 번에 모든 인스턴스를 재시작하지 않고 순차 배포가 적용되어 있는가?
- [ ] Blue-Green 배포 시 트래픽 전환 전 Green 환경이 정상인지 확인했는가?

## 배포 후 확인

### 모니터링 지표 (배포 후 10~30분)
- [ ] 에러율(5xx)이 급증하지 않는가?
- [ ] API 응답 시간이 배포 전과 유사한가?
- [ ] CPU, 메모리 사용률이 정상 범위인가?
- [ ] DB 커넥션 풀 사용률이 정상인가?

### 로그 확인
- [ ] `ERROR` 레벨 로그가 비정상적으로 증가하지 않는가?
- [ ] 스택 트레이스(NullPointerException, SQLException 등)가 새로 발생하지 않는가?
- [ ] 애플리케이션 시작 로그가 정상적으로 완료되었는가?

### 주요 기능 스모크 테스트
- [ ] 이번 배포에서 변경된 API가 정상 동작하는가?
- [ ] 핵심 사용자 플로우(로그인, 결제 등 핵심 기능)가 정상인가?

## 롤백 계획

- [ ] 롤백 기준이 사전에 정의되어 있는가? (예: 에러율 1% 초과 시 즉시 롤백)
- [ ] 이전 버전으로 롤백하는 절차가 문서화되어 있는가?
- [ ] DB 마이그레이션이 적용된 경우 롤백 시 데이터 정합성을 어떻게 처리할지 계획이 있는가?
- [ ] 롤백 담당자와 의사결정 기준이 팀에 공유되어 있는가?

## Notes

- 배포는 업무 시간 중, 팀원이 모니터링 가능한 시간대에 진행한다 (금요일 오후/공휴일 전날 배포 지양)
- 배포 후 최소 30분은 모니터링 대시보드를 주시한다
- 배포 기록(버전, 시각, 배포자, 주요 변경사항)을 채널에 공유한다
- 이상 징후 발견 시 원인 파악보다 롤백을 먼저 판단한다 (사용자 영향 최소화 우선)
