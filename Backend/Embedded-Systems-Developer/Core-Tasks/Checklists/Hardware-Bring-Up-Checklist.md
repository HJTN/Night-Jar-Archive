# Hardware Bring-Up Checklist

> 새 하드웨어 보드 최초 동작 검증 항목별 체크리스트

## 전원 인가 전 육안 검사

- [ ] 부품 방향(극성) 확인: 전해 캐패시터, 다이오드, IC 방향
- [ ] 납땜 브릿지(쇼트) 없음 확인
- [ ] 미납땜 핀 없음 확인
- [ ] VCC-GND 간 저항 측정 — 0Ω(단락)이면 전원 인가 금지

## 전원 전압 측정

- [ ] 주 전원 레일(5V/3.3V 등) 전압 측정 및 허용 범위 내 확인
- [ ] MCU VDD 핀 전압 확인 (STM32 기준: 1.71V ~ 3.6V)
- [ ] 전원 시퀀스 확인 (복수 레일 존재 시 인가 순서 준수)
- [ ] 전류 소비량 기록 (후속 비교 기준)

## MCU 부팅 확인

- [ ] 디버거(ST-Link/J-Link) 연결 후 [[OpenOCD]] 접속 성공
- [ ] `monitor reset halt` 명령으로 MCU 정지 및 접근 확인
- [ ] MCU Device ID 읽기 성공 (올바른 MCU 확인)
- [ ] [[Bootloader]] BOOT0 핀 설정 확인 (일반 실행 모드)

## 클럭 설정 검증

- [ ] `HAL_GetTick()` 1ms 단위 증가 확인 (SysTick 동작)
- [ ] 외부 클럭(HSE) 사용 시: 오실로스코프로 입력 클럭 확인
- [ ] SystemCoreClock 값이 의도한 주파수와 일치 확인

## UART 디버그 출력 확인

- [ ] [[UART]] 디버그 포트 초기화 및 시리얼 터미널 접속 확인
- [ ] `"Boot OK"` 등 초기 메시지 출력 확인
- [ ] Baud Rate 설정 일치 확인 (보통 115200)

## GPIO 핀 동작 확인

- [ ] LED Blink 테스트: 500ms 주기 토글로 [[GPIO]] 출력 동작 확인
- [ ] 버튼 입력 읽기: Pull-up 설정, 눌림/해제 정확히 감지 확인
- [ ] 회로도와 핀 번호 일치 여부 교차 확인

## 통신 인터페이스 순차 검증

- [ ] **[[I2C]]**: I2C Scanner로 예상 슬레이브 주소 응답 확인
- [ ] **[[SPI]]**: SPI 루프백(MOSI→MISO 연결) 에코 테스트
- [ ] **[[SPI]]**: 실제 Flash/디스플레이 장치와 통신 확인
- [ ] **[[UART]]** (외부 장치): 에코 테스트 및 데이터 무결성 확인
- [ ] **CAN** (해당 시): CAN 분석기로 송수신 확인
- [ ] **USB** (해당 시): PC 장치 인식 확인

## 인터럽트 동작 확인

- [ ] [[Interrupt|GPIO 외부 인터럽트]] (EXTI): 트리거 및 ISR 진입 확인
- [ ] 타이머 인터럽트: 오실로스코프로 주기 정확성 측정
- [ ] UART 수신 인터럽트: 수신 데이터 정확성 확인

## 통합 테스트

- [ ] 모든 주변장치 동시 동작 테스트
- [ ] 전원 사이클 5회 후 일관된 부팅 동작 확인
- [ ] 온도 테스트 (해당 시: 동작 온도 범위 양 끝단)

## 문서화

- [ ] 발견된 하드웨어 이슈 목록 작성 (회로 수정 필요 항목)
- [ ] 브링업 완료 날짜 및 서명 기록
- [ ] 검증된 펌웨어 기준 버전 기록

## Notes

단계별로 순서를 지켜 진행한다. 전 단계가 검증되지 않은 상태에서 다음 단계로 넘어가면 문제 원인을 찾기 어려워진다.
